# Описание итоговой IDEF0-модели

## Наименование моделируемой функции

Моделируемая функция: **«Обработать покупку цифровой игры в Steam»**.

## Контекстная диаграмма A-0

Контекстная диаграмма A-0 показывает процесс покупки цифровой игры в Steam как одну общую функцию.

| Тип дуги | Наименование | Описание |
|---|---|---|
| Input | Данные заказа | Сведения о выбранной игре, цене, аккаунте пользователя и выбранном способе оплаты. |
| Input | Выбранная игра | Цифровой товар, который пользователь намерен приобрести. |
| Input | Данные аккаунта пользователя | Информация о пользователе, необходимая для оформления покупки и добавления игры в библиотеку. |
| Input | Способ оплаты | Метод оплаты, выбранный пользователем при оформлении покупки. |
| Control | Правила Steam | Общие правила работы платформы и обработки цифровых покупок. |
| Control | Условия оплаты | Требования к проведению платежа и подтверждению операции. |
| Control | Региональные ограничения | Ограничения доступности игры в зависимости от региона аккаунта. |
| Control | Политика возвратов | Правила возврата средств при ошибке или отказе пользователя от покупки. |
| Output | Игра в библиотеке пользователя | Основной положительный результат процесса — игра становится доступна пользователю. |
| Output | Подтверждение покупки | Уведомление о том, что покупка успешно завершена. |
| Output | Сообщение об ошибке оплаты | Альтернативный результат при неуспешной оплате. |
| Mechanism | Пользователь | Инициирует покупку и подтверждает выбранные параметры заказа. |
| Mechanism | Платформа Steam | Обрабатывает заказ, проверяет данные и фиксирует результат покупки. |
| Mechanism | Платёжный сервис | Выполняет проверку и проведение оплаты. |
| Mechanism | Библиотека Steam | Хранит приобретённые пользователем игры и предоставляет доступ к ним. |

## Декомпозиция A0

Диаграмма A0 раскрывает функцию «Обработать покупку цифровой игры в Steam» через пять подфункций: A1 «Принять выбор пользователя», A2 «Сформировать заказ», A3 «Провести оплату», A4 «Добавить игру в библиотеку», A5 «Уведомить пользователя о результате». Такая логика отражает последовательное преобразование входных данных заказа в итоговый результат: доступную пользователю игру либо сообщение об ошибке оплаты.
